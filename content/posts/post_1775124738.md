In a bizarre incident that has raised eyebrows in the tech community, Anthropic, a leading AI research organization, inadvertently took down thousands of GitHub repositories, including legitimate forks of its own codebase, as part of an attempt to reclaim leaked source code. The mishap has sparked concerns about the company's ability to execute and comply with digital copyright laws, particularly as it prepares for an initial public offering (IPO).

The trouble began when a software engineer discovered that Anthropic had inadvertently included access to the source code for its popular Claude Code command-line application in a recent release. AI enthusiasts quickly pored over the leaked code, seeking insights into how Anthropic harnesses large language models (LLMs) to power its applications. As the leaked code spread across GitHub, users began sharing their findings and exploring the implications of Anthropic's approach.

In response to the leak, Anthropic issued a takedown notice under U.S. digital copyright law, asking GitHub to take down repositories containing the offending code. However, according to GitHub's records, the notice was executed against an astonishing 8,100 repositories – including legitimate forks of Anthropic's own publicly released Claude Code repository. This unintended consequence of the takedown notice sparked outrage on social media, with users whose code was inadvertently blocked by the notice expressing their frustration.

In a statement, Boris Cherny, head of Claude Code at Anthropic, acknowledged that the move was accidental and retraced the bulk of the takedown notices, limiting the impact to one repository and 96 forks containing the accidentally released source code. "The repo named in the notice was part of a fork network connected to our own public Claude Code repo, so the takedown reached more repositories than intended," an Anthropic spokesperson explained. "We retracted the notice for everything except the one repo we named, and GitHub has restored access to the affected forks."

As Anthropic prepares to go public, this botched clean-up effort serves as a sobering reminder of the importance of attention to execution and compliance in the tech industry. Leaking source code as a publicly traded company is likely to attract shareholder lawsuits, casting a shadow over the company's IPO plans. The incident also underscores the need for robust digital copyright laws and procedures that minimize collateral damage when attempting to reclaim leaked intellectual property.

Ultimately, this episode serves as a cautionary tale about the unintended consequences of code cleanup efforts. As AI research organizations continue to push the boundaries of innovation, it is essential to prioritize transparency, accountability, and careful execution in all aspects of their operations – including code management and digital copyright compliance.
